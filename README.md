# 정보처리기사 필기 학습 시스템

정보처리기사 필기 시험을 준비하는 수험생을 위한 종합 학습 웹 애플리케이션입니다.

## 사용 언어 및 기술 스택

| 언어/기술 | 비율 | 용도 |
|-----------|------|------|
| JavaScript | 87.5% | 프론트엔드 로직, 백엔드 서버, 데이터 관리 |
| HTML | 12.5% | 웹 페이지 구조 및 UI |
| CSS (Tailwind) | - | 스타일링 (CDN 사용) |

### 상세 기술 스택
- **Frontend**: Vanilla JavaScript, HTML5, Tailwind CSS
- **Backend**: Node.js, Express.js
- **Database**: JSON 파일 기반 저장소 (users.json)
- **인증**: express-session, bcryptjs

## 개발 중 겪은 오류 및 해결 방법

### 1. JavaScript 템플릿 리터럴 문법 오류
**문제**: 클릭해도 화면이 전환되지 않는 현상 발생

**원인**: `data.js` 파일에서 템플릿 리터럴의 백틱(`) 문자가 `\`` 형태로 이스케이프되어 있어 JavaScript 파싱 오류 발생. 26곳에서 동일한 오류 발견.

**해결**:
```bash
# sed 명령어로 모든 이스케이프된 백틱을 정상 백틱으로 교체
sed -i 's/\\`/`/g' data.js
```

**교훈**: 대용량 데이터 파일 작성 시 문법 검사 필수
```bash
node -c data.js  # JavaScript 문법 검사
```

### 2. 사이드바 토글 기능 미작동
**문제**: 모바일에서 사이드바가 열린 후 닫을 수 없는 문제

**원인**: 토글 버튼이 `lg:hidden` 클래스로 데스크톱에서 숨겨져 있었고, 사이드바 상태 관리 로직이 불완전

**해결**:
- 토글 버튼을 모든 화면 크기에서 표시
- CSS에서 사이드바 트랜지션 로직 개선
- JavaScript에서 화면 크기별 동작 분기 처리

### 3. D-Day 고정값 문제
**문제**: D-Day가 항상 "D-57"로 표시

**원인**: HTML에 하드코딩된 값만 있고, JavaScript에서 동적 계산이 없음

**해결**: `updateDDay()` 함수 구현
```javascript

```

### 4. 포트 충돌 (EADDRINUSE)
**문제**: 서버 재시작 시 "address already in use :::3000" 오류

**원인**: 이전 Node.js 프로세스가 종료되지 않고 포트 점유

**해결**:
```bash
# 포트 사용 프로세스 확인
netstat -ano | grep 3000

# 프로세스 강제 종료
taskkill /F /PID [프로세스ID]
```
